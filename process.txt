Процессы:

Процесс: создание нового аккаунта. После валидации формы создается запись в таблице User. Происходит  переадресация пользователя на страницу списка заданий.
Процесс: добавление нового задания. Создается запись в таблице task c статусом «new». При загрузке аттачментов, файлы сохраняются в директории пользовательских файлов (web/uploads/{{user_id}}), а путь сохраняется в связанной таблице task_attachment
Процесс: отклик на задание. Только авторизованному пользователю и только ранее не откликавшемуся на это задание (уникальная пара user_id, task_id в таблице responses). Задание  с статусом  «new». Добавляется запись в таблицу Responses. Отправляется уведомление на почту автора объявления.
Процесс: выбор исполнителя. Только автору объявления с статусом  «new». В  соответствующей записи таблицы tasks меняется значение executor_id.  Tasks.status = proccess. Отправляется уведомление на почту выбранного исполнителя.
Процесс: завершение задания. Только авторизованному пользователю и только автору задания в статусе «process». Создается запись в таблице ratings c привязкой к таблице responses.  Tasks.status = success. Отправляется уведомление на почту выбранного исполнителя.

Процесс: отказаться от задания. Только авторизованному пользователю и выбранному в качестве исполнителя. Добавляется запись в таблице ratings c привязкой к таблице responses c оценкой 0. Для счетчика проваленных заданий возможно выбирать записи с оценкой 0. Tasks.status = fail. Отправляется уведомление на почту автора объявления.
